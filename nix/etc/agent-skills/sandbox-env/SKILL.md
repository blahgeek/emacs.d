# 在sandbox外运行命令

### 什么时候适用

1. 一些需要认证的命令，比如`kubectl`, `ssh` 等。在sandbox中可能没有相应的auth文件或环境变量或者权限受限，但sandbox外有。（不过请总是先在sandbox内尝试运行，明确没权限再继续。）
2. 需要单次访问sandbox外的文件，或者单次写入sandbox内只读的文件
3. 需要操作一些系统资源

### 使用方式

```bash
sandbox-run-outside CMD...
```

比如 `sandbox-run-outside ps aux`。用法类似`sudo`，就是加一个前缀。

调用后，这个命令会向用户请求确认，用户确认后将会在sandbox外运行命令。命令的stderr和stdout行为与原来完全一致，return code也会被保留，运行的working directory也会被正确设置给命令。

如果用户没有批准这个运行，命令将直接退出并有相应的错误信息。这种情况下一定不能重试，而是直接等待用户进一步的指令。


# 把sandbox中一个目录或文件变成可写

### 什么时候适用

当你需要多次编辑$HOME目录内的某个项目中的文件，但发现当前是只读的。你可以请求把该目录或文件变成可写。

注意：

1. 这个目录或文件必须已经存在
2. 合理设置可写的粒度：一般情况下，设置整个项目的目录即可。不要请求一个很大的范围，比如整个$HOME，或者~/work；也不要请求一个很深的子文件夹。

### 使用方式

```
sandbox-set-writable DIR
```

比如`sandbox-set-writable ~/Code/myproject` ，这会请求用户把 ~/Code/myproject 目录从只读变成可写。

调用后，这个命令会向用户请求确认。如果用户没有批准这个运行，命令将直接退出并有相应的错误信息。这种情况下一定不能重试，而是直接等待用户进一步的指令。
